# Phase 1 Eval Labels

This folder keeps labels separate from model outputs.

In simple terms: the model is the student, and this file becomes the answer key only when humans source-check the labels. Do not let the student write the answer key.

## Files

- `phase1_labels.jsonl` - one label per event.
- `phase1_codex_source_reviews.jsonl` - one Codex source-review summary per non-human-reviewed label.
- `phase1_report.json` - generated by `bun run eval:phase1`.

## Label Rules

Use `labelSource: "human"` and `humanReviewed: true` only after a person has reviewed the row.

For a label to count toward a model-improvement claim, `reviewContext.sourceChecked` must also be `true`. That means the reviewer opened the source URL or inspected enough source context to judge the real event.

Use `labelSource: "llm_article_review"` for labels generated by `bun run review:phase1:llm`. These labels read the source article text when available, but they are still model-generated.

The current Codex triage pass can be reapplied with `bun run review:phase1:codex`. It uses an intelligence-analysis lens and writes `labelSource: "llm_article_review"`, not human ground truth.

The Codex source-review pass can be reapplied with `bun run review:phase1:codex-source`. It writes article/source summaries and Codex rationales into `reviewContext`, preserves human-reviewed labels, and still writes `labelSource: "llm_article_review"` for machine-reviewed rows.

Use `labelSource: "bootstrap_current_rules"` only for starter rows generated from current event fields. These rows prove the eval wiring works, but they do not prove the model improved.

Phase 1 requires at least 100 source-checked human labels before the report can say the candidate model improved.

## Human Review

List the review queue:

```bash
bun run review:phase1:human -- --list
```

Review labels interactively:

```bash
PHASE1_REVIEWER=your-name bun run review:phase1:human
```

Fast review prior LLM/Codex labels after checking the source URL or enough source context:

```bash
PHASE1_REVIEWER=your-name bun run review:phase1:human -- --fast
```

Optional controls:

- `PHASE1_HUMAN_REVIEW_LIMIT=20` reviews only 20 rows.
- `--limit=20` reviews or lists only 20 rows.
- `--fast` shows one-key accept/edit/skip controls for prior labels, but accepted labels still require source/context confirmation.
- `--force` includes rows that are already human-reviewed.

After reviewing, rerun:

```bash
bun run eval:phase1
```

## LLM Review

Run:

```bash
bun run review:phase1:llm
bun run eval:phase1
```

This requires `ANTHROPIC_API_KEY`.

Optional controls:

- `PHASE1_REVIEW_LIMIT=20` reviews only 20 rows.
- `PHASE1_REVIEW_FORCE=1` re-reviews rows that already have LLM labels.
- `ANTHROPIC_REVIEW_MODEL=claude-3-5-haiku-20241022` chooses the review model.
